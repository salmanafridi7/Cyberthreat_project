About This Project

Understanding cyber threats through a Flask-based intelligence dashboard.

Project Overview

This project is a Cyber Threat Intelligence Dashboard developed using Flask, SQLite, SQLAlchemy, HTML, and CSS. The dashboard stores and displays cybersecurity threat intelligence records using a database-driven web application.

What is Cyber Threat Intelligence?

Cyber Threat Intelligence is information collected about cyber attacks, vulnerabilities, suspicious activity, malicious domains, harmful IP addresses, and other online threats.

Organizations use threat intelligence to monitor risks, detect attacks early, improve cybersecurity awareness, and strengthen digital security.

About the Dataset

The dataset used in this project contains cybersecurity-related threat records collected from open threat intelligence sources such as OTX (Open Threat Exchange).

These records may include malicious IP addresses, suspicious domains, vulnerabilities, malware indicators, phishing activity, and other cybersecurity-related information.

What are the Threat IDs?

Each threat record contains a unique identifier. These IDs are used to uniquely identify and track threat intelligence records inside databases and security monitoring systems.

In real-world cybersecurity platforms, unique threat IDs help analysts search, investigate, and reference specific threats efficiently.

What are Malicious IP Addresses?

A malicious IP address is an internet address associated with suspicious or harmful activity such as malware distribution, phishing attacks, spam campaigns, botnets, or unauthorized access attempts.

Security teams monitor malicious IPs to block dangerous traffic and reduce cybersecurity risks.

What are Malicious Domains?

Malicious domains are websites or domain names connected to phishing, malware downloads, fake login pages, scams, or other harmful online activities.

Monitoring suspicious domains helps organizations prevent users from accessing unsafe websites.

What are CVEs?

CVE stands for Common Vulnerabilities and Exposures. CVEs are publicly disclosed cybersecurity vulnerabilities found in software, applications, operating systems, or network systems.

Security professionals use CVEs to identify weaknesses and apply security updates or patches before attackers exploit those vulnerabilities.

What is OTX?

OTX (Open Threat Exchange) is a cybersecurity threat intelligence platform where security researchers and organizations share information about cyber threats, malware indicators, suspicious domains, attack patterns, and malicious activity.

Why This Dashboard is Useful

  • Helps organize cybersecurity threat data.
  • Allows users to search and browse threat records.
  • Displays cybersecurity data dynamically using Flask.
  • Demonstrates database integration using SQLite and SQLAlchemy.
  • Provides a simplified example of a cyber threat monitoring system.
  • Shows how real-world cybersecurity datasets can be visualized through web applications.

Technologies Used

Frontend: HTML and CSS

Backend: Python Flask

Database: SQLite

ORM: SQLAlchemy

Dataset Type: Cyber Threat Intelligence Data

Project Objective

The objective of this project is to demonstrate how cybersecurity data can be imported, stored, queried, and displayed using a Flask-based database-driven web application.
